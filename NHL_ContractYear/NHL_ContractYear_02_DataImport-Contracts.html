NHL Contract Year Effects: Player Performance and Expectations (2014/2015-2024/2025)

Adam Morgenstern
Last updated: 2026-02-05

1. Overview

Obtaining reliable historical NHL contract data proved more challenging than expected. Most major sources do not offer free public APIs, and some only included date on current active players. Sportrac, however, while not providing an API, does include enough data to effectively reconstruct which players were in a contract year for each NHL season.

Sportrac offers three useful tables, which we can filter by season:

  • Signed Free Agents: players who were scheduled to become free agents that offseason, but were signed by a team sometime after the season.
  • Available Free Agents: players who reached the end of their contract and unsigned.
  • Extensions: players who signed contract extensions during the season.

For the purposes of this study:
Player who signed a new contract or extension during a season will still be considered to have been in a contract year, even if the signing occured mid-season. These players will likely have played at least some games with an expiring contract.
Players who signed a contract extension before the start of the season will not be considered to have been in a contract year, since they will not have played any games that season with an expiring contact.
Sportrac's date filters allow us to identify these early signers.

2. Data Organization

All Sportrac data will be manually copied into Google Sheets.
Each NHL season will have a separate sheet containing:

  • Signed Free Agents
  • Available Free Agents
  • Contract Extensions

A Season column will be added to each table in the YYYYYYYY (20192020) format to match our NHL API Stats dataset.
Because Spotrac only includes Free Agent Type (Restricted/Unrestricted) in the *Available table, this data will be missing for our other sources. We'll include the column for now with the expectation that it may be completed later on.

A separate Early Signers sheet will include all players who signed an extension between July 1st and the day before the season began for each year. We will later filter these players out if they appeared in the contract year sheet for that season.
Spotrac only lists names in a "First Last (POS)" column for the Extensions table, so we will use two formulas to split this into new Player Name and Position columns:

=TRIM(REGEXREPLACE(B2, "\s*\(.*\)", ""))  
=REGEXEXTRACT(B2, "\((.*)\)")


These clean names will allow us to match early signers with the combined contract year dataset later on.

4.1 Player Name Correction

After identifying more than 1,000 player names appearing in contract_years but not in skater_season_stats, we investigated the source of these mismatches. Manual inspection of several examples revealed two main categories:

1. Players who never appeared in the NHL during our timespan despite signing an NHL contract

  • Example: Luke Walker signed with Colorado but never played an NHL game
  • These cases are not relevant to our statistical analysis and can be safely ignored

2. Players whose names differ slightly between datasets

  • Example: Pat Dwyer / Patrick Dwyer
  • These require correction so their season data can be properly joined to their contract records

To detect and correct these discrepancies, we will perform the following steps:

  • Export the unmatched player list from BigQuery
  • Export the list of all unique player names from skater_season_stats
  • Run fuzzy matching in Python with RapidFuzz, retrieving the top three closest skater_season_stats matches for each unmatched contract_year name
  • Import the results into Google Sheets, sort by the match score, and manually select the correct match (or mark as unmatched) in the correct_match column
  • Apply corrections in the original Google Sheets files prior to re-uploading the cleaned datasets to BigQuery

from rapidfuzz import process, fuzz
import pandas as pd

df_contract = pd.read_csv("Data/contractYear_unmatchedPlayerNames.csv")
df_nhl = pd.read_csv("Data/nhl_stats_uniquePlayerNames.csv")

nhl_names = df_nhl["playerName"].tolist()

results = []

# Find the top 3 matches for each unmatched name

for name in df_contract["playerName"]:
    matches = process.extract(
        name,
        nhl_names,
        scorer=fuzz.token_sort_ratio,
        limit=3
    )

    # Fill with blanks if fewer than 3 matches
    while len(matches) < 3:
        matches.append(("", 0.0, 0))

    row = {
        "contract_name": name,
        "nhl_name_1": matches[0][0],
        "match_score_1": matches[0][1],
        "nhl_name_2": matches[1][0],
        "match_score_2": matches[1][1],
        "nhl_name_3": matches[2][0],
        "match_score_3": matches[2][1],
        "correct_match": ""           # fill in later manually
    }

    results.append(row)

df_results = pd.DataFrame(results)
df_results.to_csv("Data/name_fuzzy_matches_top3.csv", index=False)

4.2 Removing Early Signers

With the updated contract_years and early_signers tables imported back into SQL, we can proceed.
We'll remove the early_signers from the contract_years by matching playerName and season and creating a new table, contract_years_no_early.

CREATE OR REPLACE TABLE `nhl_contract_project.contract_years_no_early` AS
SELECT *
FROM `nhl_contract_project.contract_years` c
WHERE NOT EXISTS (
  SELECT 1 
  FROM `nhl_contract_project.early_signers` e 
  WHERE c.playerName = e.playerName 
    AND c.season = e.season
)
ORDER BY season ASC, playerName ASC;

SELECT 
  (SELECT COUNT(*) FROM `nhl_contract_project.contract_years`) - 
  (SELECT COUNT(*) FROM `nhl_contract_project.contract_years_no_early`) 
AS records_removed;

486 early_signer rows were removed from contract_years in the new table.

4.3 Adding the Contract Year Column

Now that we've created the contract_years_no_early table as our finalized dataset of player contract years, we'll add a new boolean column to the skater_season_stats table, and use that new table to fill it out.
We'll also add a free_agent_type column and fill it with the partial fa_type data, since we may be able to add to this and use it at some point.
The result will be saved as a new table skater_season_stats_with_contract.

CREATE OR REPLACE TABLE `nhl_contract_project.skater_season_stats_with_contract` AS
SELECT 
    s.*,
    CASE 
        WHEN c.playerName IS NOT NULL THEN TRUE
        ELSE FALSE
    END AS contract_year,
    c.fa_type AS free_agent_type
FROM `nhl_contract_project.skater_season_stats` s
LEFT JOIN `nhl_contract_project.contract_years_no_early` c
    ON s.playerName = c.playerName
    AND s.season = c.season;
